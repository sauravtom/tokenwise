The problem it solves

Without a tool to read your code, AI assistants guess. yoyo stops the guessing.

Without yoyo

  • Wrong file pathsAI assistants invent plausible-sounding paths that don't exist in your project.
  • Stale function namesAnswers come from training data, not your current code on disk.
  • 20-minute onboardingTo understand a 400-file project, the AI reads dozens of files one by one.
  • Missed call chainsWithout a call graph, the AI can't tell you what breaks when you rename a function.

With yoyo

  • Exact file paths and line numbersEvery answer comes from the AST. File, line, and byte offset are exact.
  • Current function namesyoyo reads your files on disk. The index is always in sync after every edit.
  • 30-second onboarding8 parallel tool calls replace 15–20 minutes of directory browsing and grepping.
  • Full call graphblast_radius finds every function that calls yours, transitively.

What it does

yoyo runs one command to parse your project, then serves the results over MCP.

Parses your code with ast-grep

Builds an index of every function, its file, line range, byte offsets, complexity score, and outgoing calls. Supports 14 languages: TypeScript, JavaScript, Rust, Python, Go, C, C++, C#, Java, Kotlin, PHP, Ruby, Swift, and Bash.

Gives Claude exact answers

symbol returns the file, line number, and optionally the full function body. supersearch finds every call site across hundreds of files. No guessing.

Traces call chains

blast_radius finds everything that calls a function, transitively. trace_down follows a function to its database, HTTP, or queue boundary (Rust + Go).

Edits code safely

patch writes changes by symbol name, line range, or exact string match. Validates syntax after every write. Reindexes automatically so the next read is always fresh.

Audits dead code

health reports functions that are never called, functions with high complexity and high fan-out, and functions with duplicate names across files. Public API is excluded — only genuinely unreachable code is flagged.

Semantic search — no API key

semantic_search finds functions by intent: "semaphore acquisition", "spawn blocking task", "async runtime builder". Powered by local ONNX embeddings (fastembed AllMiniLML6V2). No network, no key.

Runs entirely on your machine

Single static binary. No network requests, no API keys, no telemetry. The index is a plain JSON file in your project root.

Installation

One binary, no runtime dependencies.

Pre-built binary (recommended)

# macOS Apple Silicon
curl -L https://github.com/avirajkhare00/yoyo/releases/latest/download/yoyo-aarch64-apple-darwin.tar.gz \
  | tar xz
sudo mv yoyo-aarch64-apple-darwin /usr/local/bin/yoyo
# Linux x86_64
curl -L https://github.com/avirajkhare00/yoyo/releases/latest/download/yoyo-x86_64-unknown-linux-gnu.tar.gz \
  | tar xz
sudo mv yoyo-x86_64-unknown-linux-gnu /usr/local/bin/yoyo

Build from source

git clone https://github.com/avirajkhare00/yoyo.git
cd yoyo && cargo build --release
sudo cp target/release/yoyo /usr/local/bin/yoyo

Quick start

# index your project
yoyo bake --path /path/to/your/project

# overview: languages, file counts, most complex functions
yoyo shake --path /path/to/your/project

# find a function by name
yoyo symbol --path /path/to/your/project --name myFunction

# search across all files
yoyo supersearch --path /path/to/your/project --query myFunction
