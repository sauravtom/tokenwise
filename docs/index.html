Why yoyo?

Real strengths drawn from benchmarks on production codebases.

⚡

Instant onboarding

Run bake + shake and get the top-complex functions, language breakdown, and project structure in seconds — no reading required.

🔍

Symbol-level search

symbol finds any function by name; use --include-source to get the body inline. slice reads exact line ranges. Jump to a 260-line function with one command, no IDE needed.

🕸️

Call-chain tracing

trace_down follows a function's call chain to db/http/queue boundaries (Go + Rust). supersearch finds every usage across hundreds of files in one query.

📦

Module deep-dives

package_summary surfaces the full function list and complexity map of any module — without opening a single file.

🗺️

Placement guidance

suggest_placement and architecture_map score candidate files for new code by path heuristics and proximity to related symbols.

🔒

Fully local & private

No API keys, no SaaS, no telemetry. Your code never leaves your machine. The bake index is a plain JSON file in your project root.

🤖

AI-native (MCP)

Connect to Claude Code, Cursor, or any MCP-compatible assistant. All 22 tools are available over stdio — grounding the AI in real code, not guesses.

🦀

Pure Rust performance

Single static binary. No Node, no Python, no runtime. Ships as a pre-built binary for Linux x86_64 and macOS ARM64.

Installation

Download a pre-built binary or build from source with Cargo.

Option 1 — Pre-built binary (recommended)

Download from the latest GitHub release:

# macOS (Apple Silicon)
curl -L https://github.com/avirajkhare00/yoyo/releases/latest/download/yoyo-aarch64-apple-darwin.tar.gz \
  | tar xz
sudo mv yoyo-aarch64-apple-darwin /usr/local/bin/yoyo
# Linux (x86_64)
curl -L https://github.com/avirajkhare00/yoyo/releases/latest/download/yoyo-x86_64-unknown-linux-gnu.tar.gz \
  | tar xz
sudo mv yoyo-x86_64-unknown-linux-gnu /usr/local/bin/yoyo

Option 2 — Build from source

# requires Rust stable (https://rustup.rs)
git clone https://github.com/avirajkhare00/yoyo.git
cd yoyo
cargo build --release
sudo cp target/release/yoyo /usr/local/bin/yoyo

Quick start

# 1. index your project
yoyo bake --path /path/to/your/project

# 2. get an instant overview
yoyo shake --path /path/to/your/project

# 3. find a function by name
yoyo symbol --path /path/to/your/project --name myFunction

# 4. search across all files
yoyo supersearch --path /path/to/your/project --query prisma

Use with AI assistants (MCP)

Connect yoyo to Claude Code, Cursor, or any MCP-compatible assistant.

MCP server mode

Run yoyo as a stdio MCP server. The AI can call all 22 tools directly — grounding its answers in your actual code instead of guessing at structure.

Add this to your ~/.claude/settings.json (Claude Code) or Cursor MCP config:

{
  "mcpServers": {
    "yoyo": {
      "type": "stdio",
      "command": "/usr/local/bin/yoyo",
      "args": ["--mcp-server"]
    }
  }
}
